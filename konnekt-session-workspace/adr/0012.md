# ADR-0012: Use garde for Domain Validation

**Status**: Accepted | **Date**: 2025-12-29

## Context

Need declarative validation for participant names (length, chars), lobby capacity, activity data, commands. WASM-compatible — no heavy regex engines.

## Decision

Use **`garde`** for derive-based domain validation.

```rust
#[derive(Debug, Validate)]
pub struct Participant {
    #[garde(length(min = 1, max = 50))]
    pub name: String,
}

participant.validate()?;
```

## Why

- Derive macro — validation rules are documentation
- ~15KB WASM overhead (vs `validator`'s ~100KB with `fancy_regex`)
- Custom validators: `#[garde(custom(validate_unique_names))]`
- Structured error reports: `"participants[0].name: length is lower than 1"`
- Integrates with `thiserror` via `#[from] garde::Report`

## Rules

- ✅ Validate at system boundaries (commands, value objects)
- ❌ Don't re-validate already-constructed internal state

## Trade-offs

- ✅ Declarative, minimal boilerplate, WASM-friendly
- ❌ Less mature than `validator` crate, runtime-only validation
