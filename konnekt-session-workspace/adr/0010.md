# ADR-0010: Event Sourcing for Lobby State

**Status**: Accepted | **Date**: 2025-12-29

## Decision

Lobby state is derived from an append-only event log. Events are signed by the host and broadcast to all peers.

## Why

- Full audit trail of all lobby state changes
- Enables state reconciliation after reconnect
- Peers can replay events to reach consistent state

## Trade-offs

- Event log grows over time (mitigated by snapshots)
- Requires careful event versioning
