# ADR-0004: Client-Side Key Generation for Persistent Identity

**Status**: Accepted | **Date**: 2025-12-29

## Decision

Derive Ed25519 key pairs client-side from name + password. Store in `localStorage`.

## Why

- Deterministic: same credentials → same key → reconnection works
- No server-side identity store needed
- Enables [[../concepts/host-delegation|host reclaim]] after disconnect

## Trade-offs

- Key loss = identity loss (mitigate with export/backup UI)
- Password change = new identity
