# ADR-0023: Use tracing-chrome for Performance Profiling

**Status**: Accepted | **Date**: 2025-12-30

## Context

Need to measure P2P latency, identify bottlenecks, visualize execution timelines. Traditional tools (perf, flamegraph) don't integrate with `tracing` spans.

## Decision

Use **`tracing-chrome`** to export tracing spans to Chrome's trace format, viewable in **Perfetto**.

```toml
tracing-chrome = { version = "0.7", optional = true }

[features]
chrome-trace = ["tracing-chrome"]
```

```bash
cargo run --features chrome-trace -- create-host --name Alice
# → writes trace.json
# Open https://ui.perfetto.dev/ and drag-drop trace.json
```

## What You Get

```
p2p_loop::poll           ████████      10ms
  ├─ poll_events         ███            3ms
  └─ translate_messages  ██             2ms

domain_loop::poll                █████  5ms
  └─ process_commands            █████  5ms
```

## tokio-console vs tracing-chrome

| Aspect | tokio-console | tracing-chrome |
|--------|---------------|----------------|
| Use | Live debugging | Post-mortem profiling |
| View | Real-time TUI | Timeline in Perfetto |
| Best for | Deadlocks, starvation | Latency, bottlenecks |

Use **both together** for full picture: `--features "console,chrome-trace"`.

## Trade-offs

- ✅ Visual timeline, nested spans, ~5-10% overhead
- ❌ Large files for long sessions, post-mortem only
