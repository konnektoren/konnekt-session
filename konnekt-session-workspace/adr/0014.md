# ADR-0014: Use localStorage for Session Persistence

**Status**: Accepted | **Date**: 2025-12-29

## Context

Users lose identity on page refresh. Need to persist session ID (key derivation seed), last lobby ID, preferences — without storing secrets.

## Decision

Use **`gloo-storage`** wrapping `localStorage`.

## What Gets Stored

| Stored ✅ | NOT Stored ❌ |
|-----------|--------------|
| Session ID (UUID) | Private keys |
| Last lobby ID | Passwords |
| User preferences | Full lobby state |

Keys are re-derived on demand from `session_id + name + optional_password`.

## Why

- Browser-native, persistent across refresh/restart
- Synchronous API — no async overhead for small data
- gloo abstraction handles serialization + errors
- `<100KB` data fits well within 5MB limit

## Trade-offs

- ✅ User keeps identity on refresh, simple API
- ❌ User can clear storage (acceptable — creates new session)
- ❌ 5MB total quota shared across domain
