# ADR-0009: Use JSON with Feature-Gated MessagePack

**Status**: Accepted | **Date**: 2025-12-29

## Context

P2P messages, localStorage, debug logs. Need human-readable debugging vs compact wire format. 1–10 messages/second during gameplay.

## Decision

**JSON** as default via `serde_json`. **MessagePack** via `rmp-serde` as opt-in feature.

```toml
serde_json = "1.0"
rmp-serde = { version = "1.3", optional = true }

[features]
messagepack = ["rmp-serde"]
```

## Why JSON First

- Visible in browser DevTools Network tab — instant debug insight
- Natural fit for `localStorage` (string API)
- Universal tooling (Charles, curl, mocks)
- `serde_json` is most tested, minimal WASM overhead (~20KB)

## Why MessagePack Optional

- ~35–46% size reduction for typical lobby states
- `konnekt-session = { version = "0.4", features = ["messagepack"] }` for prod
- Format negotiation via `P2PMessage` envelope

## Size Comparison

| Message | JSON | MessagePack |
|---------|------|-------------|
| Lobby (2 participants) | 245 B | 156 B |
| 10 participants | 1.2 KB | 650 B |

## Trade-offs

- ✅ Easy debugging, universal tooling, incremental optimization
- ❌ JSON ~35% larger, two formats to test
