# ADR-0015: Use Feature-Gated Event Cache

**Status**: Accepted | **Date**: 2025-12-29

## Context

Event Sourcing ([[0010|ADR-0010]]) keeps a bounded log (max 100 events, ~15KB). Need to cache for fast reconnection, offline resume, and debugging — without IndexedDB complexity.

## Decision

Use **`gloo-storage`** (localStorage) in browser, **in-memory HashMap** in tests, behind an `EventCache` trait.

```rust
pub trait EventCache {
    fn save_events(&self, lobby_id: &LobbyId, events: &[LobbyEvent]) -> Result<()>;
    fn load_events(&self, lobby_id: &LobbyId) -> Result<Vec<LobbyEvent>>;
    fn clear(&self, lobby_id: &LobbyId) -> Result<()>;
}
```

- Browser: `BrowserEventCache` → `LocalStorage`
- Tests: `InMemoryEventCache` → `HashMap`

## Why localStorage Over IndexedDB

- Bounded data: 100 events × ~150 bytes = ~15KB — well within 5MB limit
- Synchronous API — simpler, no async ceremony
- Same `EventCache` interface works for both targets

## Trade-offs

- ✅ Simple API, sufficient capacity, synchronous, testable
- ❌ Not queryable (fine — we just load all events)
- ❌ Synchronous blocks thread (~1ms for 15KB, acceptable)
