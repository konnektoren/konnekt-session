# ADR-0018: Use Ratatui for Interactive CLI Development

**Status**: Accepted | **Date**: 2025-12-29

## Context

Basic CLI ([[0016|ADR-0016]]) lacks interactivity: no real-time updates, no state inspection, hard to correlate events across terminals. Development tool — not for end users.

## Decision

Use **Ratatui** (successor to tui-rs) for an interactive terminal UI.

## Features

- **Tabs**: Lobby, Events, Participants, Activities
- **Real-time**: events appear as they arrive from P2P
- **Multi-peer**: run 3 instances in split terminal, see synchronized state
- **Keyboard**: `j/k` scroll, `q` quit, `Tab` switch

## Architecture

```
Event Loop (tokio::select!)
  ├── P2P events  → update app state
  └── Keyboard    → navigate / quit
      └── terminal.draw() → render immediately
```

## Why

- Immediate-mode rendering: pure function over app state (like React)
- `tokio::select!` integrates naturally
- Successor to tui-rs — active maintenance, used by gitui, kdash
- ~50KB binary size increase
- Terminal-native: any terminal, remote SSH

## Trade-offs

- ✅ Real-time debugging, multi-peer visibility, interactive exploration
- ❌ More complex than `println!`, terminal-only, dev tool only
