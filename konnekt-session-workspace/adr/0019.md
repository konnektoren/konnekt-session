# ADR-0019: Use Arboard for Clipboard Integration in TUI

**Status**: Accepted | **Date**: 2025-12-29

## Context

TUI ([[0018|ADR-0018]]) displays session IDs that users must share. Manual terminal text selection is inconsistent in raw mode. Need one-key copy.

## Decision

Use **`arboard`** for cross-platform clipboard access.

```rust
use arboard::Clipboard;

pub fn copy_session_id(&mut self) -> Result<(), String> {
    let mut clipboard = Clipboard::new()?;
    clipboard.set_text(&self.session_id)?;
    Ok(())
}
```

Keyboard shortcuts: `y` = copy session ID, `c` = copy full join command.

## Why

- Cross-platform: macOS (NSPasteboard), Linux X11/Wayland, Windows Win32
- Actively maintained (1Password team, v3.4)
- ~30KB binary size increase
- Feature-gated: `features = ["tui"]` — not included in core/web builds
- Graceful degradation: shows error if clipboard unavailable (headless)

## Trade-offs

- ✅ One-key copy, consistent UX, pure Rust
- ❌ Headless systems have no clipboard (expected limitation for TUI)
