# ADR-0002: Use Yew for Frontend Framework

**Status**: Accepted | **Date**: 2025-12-29

## Context

After choosing Rust ([[0001|ADR-0001]]), need a browser UI framework. Requirements: component-based, reactive state, type-safe props, hooks/composability, zero FFI overhead.

## Decision

Use **Yew** as the frontend framework.

## Why

- React-like component model familiar to web developers
- Modern hooks (`use_state`, `use_callback`, `use_context`)
- Direct access to domain types — no serialization overhead at UI boundary
- Mature ecosystem: `yew-router`, active community
- Built specifically for WASM

## Trade-offs

- ✅ Type safety through entire UI layer, single language
- ❌ Smaller ecosystem than React/Vue, limited third-party components
- ❌ Recompile required for UI changes (slower iteration)

## Migration Path

Domain logic stays in Rust. If Yew is replaced, expose WASM API via `wasm-bindgen` and swap UI layer only.
