# ADR-0022: Use tokio-console for Async Runtime Debugging

**Status**: Accepted | **Date**: 2025-12-30

## Context

Dual event loops ([[0020|ADR-0020]]) are hard to debug: task starvation, channel deadlocks, slow polls. `println!` debugging is insufficient.

## Decision

Use **tokio-console** for live async task inspection. Feature-gated, development only.

```toml
console-subscriber = { version = "0.4", optional = true }

[features]
console = ["console-subscriber", "tokio/tracing"]
```

```bash
# Terminal 1
TOKIO_CONSOLE=1 RUSTFLAGS="--cfg tokio_unstable" \
  cargo run --features console -- create-host --name Alice

# Terminal 2
tokio-console
```

## What It Shows

- Running tasks with CPU time and poll count
- Channel capacity and backpressure (spot bottlenecks)
- Blocked/waiting tasks (detect deadlocks)

## When to Use

| Use | Don't Use |
|-----|-----------|
| Task starvation | Production monitoring |
| Channel backpressure | CPU profiling (use flamegraph) |
| Async perf profiling | Memory debugging |

## Trade-offs

- ✅ Live visibility, deadlock detection, official Tokio tooling
- ❌ Requires `--cfg tokio_unstable` flag
- ❌ TCP port :6669, dev-only
