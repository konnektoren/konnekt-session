# ADR-0016: Use Clap for CLI Tooling

**Status**: Accepted | **Date**: 2025-12-29

## Context

Need a minimal CLI to test P2P logic without browser overhead. Target: `create`, `join`, `events` subcommands, live event display.

## Decision

Use **Clap v4** with derive macros.

```bash
cargo run --bin konnekt-cli create --name "Alice"
cargo run --bin konnekt-cli join --lobby abc123 --name "Bob"
cargo run --bin konnekt-cli events --lobby abc123
```

## Why

- Derive macros: minimal boilerplate, type-safe args
- Auto-generated `--help`
- Industry standard (used by Cargo, Rustup)
- Native-only: binary size doesn't matter (not WASM)

## Use Cases

- **Development**: test domain logic before building Yew UI
- **CI/CD**: automated P2P connection tests in terminal
- **Debugging**: two side-by-side terminals to see event sync

## Trade-offs

- ✅ Fast prototyping, CI/CD integration, type-safe args
- ❌ ~500KB binary size (acceptable for dev tool)
- ❌ Will be supplemented by Cucumber tests for full scenarios
