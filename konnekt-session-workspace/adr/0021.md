# ADR-0021: Use Mockall for Test Doubles

**Status**: Accepted | **Date**: 2025-12-30

## Context

Hexagonal architecture ([[0020|ADR-0020]]) has ports (traits) at boundaries. Testing domain in isolation requires mocking infrastructure ports without real network/storage.

## Decision

Use **mockall** for generating test doubles from traits.

```rust
#[cfg_attr(test, mockall::automock)]
pub trait P2PPort {
    fn broadcast(&mut self, msg: Vec<u8>) -> Result<()>;
    fn poll_events(&mut self) -> Vec<ConnectionEvent>;
}

// In tests:
let mut mock_p2p = MockP2PPort::new();
mock_p2p.expect_broadcast()
    .times(1)
    .returning(|_| Ok(()));
```

## Why

- 1.5M downloads/month — most popular Rust mocking library
- Powerful assertions: `.times(2..=5)`, `.in_sequence()`, `.withf(|arg| ...)`
- Compile-time safety: errors if mock doesn't match trait signature
- Test-only: `#[cfg_attr(test, ...)]` — zero production overhead

## Anti-Patterns

- ❌ Don't mock domain entities — they should be real instances
- ❌ Don't over-mock simple types (UUID, Instant)
- ✅ Mock only at architectural boundaries (ports/adapters)

## Trade-offs

- ✅ Fast isolated tests, behavior verification, maintainable
- ❌ ~1–2s compile overhead from proc macros
- ❌ Learning curve for mockall API
